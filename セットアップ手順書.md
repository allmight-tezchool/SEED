# セットアップ手順書（RYOさん向け）

このツールを社内で使えるようにする手順です。
所要時間: **約18分**。プログラミング知識は不要です。

---

## 全体の流れ

```
1. GitHubに新しいリポジトリを作る  ←  3分
2. このフォルダのファイルをアップロード  ←  3分
3. Vercelにサインアップしてリポジトリと連携  ←  5分
4. Claude APIキーをVercelに登録（再デプロイ含む）  ←  3分
5. 公開URLが発行される（完成）  ←  待つだけ
```

**事前に準備するもの**:
- GitHubアカウント
- Claude APIキー（`sk-ant-` で始まる文字列）

---

## ステップ1: GitHubに新しいリポジトリを作る

1. ブラウザで https://github.com にログイン
2. 画面右上の **「+」** ボタン → **「New repository」** をクリック
3. 入力する内容:
   - **Repository name**: `serp-analyzer`（好きな名前でOK）
   - **Description**: `競合サイトのGoogle流入数自動推定`（任意）
   - **Private** を選択（社内ツールなので非公開推奨）
   - **「Add a README file」** はチェック**しない**（あとで入れます）
4. 一番下の **「Create repository」** をクリック

→ 空のリポジトリができます

---

## ステップ2: ファイルをアップロード

1. 作ったリポジトリの画面で、**「uploading an existing file」** のリンクをクリック
   - もしくは「Add file」→「Upload files」
2. このフォルダ（`serp-analyzer`）の**中身を全部**ドラッグ&ドロップ
   - 含まれるもの:
     - `api/analyze.js`
     - `public/index.html`
     - `package.json`
     - `vercel.json`
     - `.gitignore`
     - `README.md`
     - `セットアップ手順書.md`（このファイル自体）
   - ⚠️ フォルダごとではなく**中身**をアップロードしてください
3. 下にスクロールして **「Commit changes」** をクリック

→ ファイルがGitHubに上がります

---

## ステップ3: Vercelに登録・連携

### 3-1. Vercelにサインアップ

1. ブラウザで https://vercel.com を開く
2. 右上の **「Sign Up」** をクリック
3. **「Continue with GitHub」** を選択
4. GitHubの認証画面が出るので **「Authorize Vercel」** をクリック

→ VercelとGitHubが連携されます（無料プランで進めばOK、クレジットカード登録不要）

### 3-2. プロジェクトをインポート

1. Vercelの管理画面で **「Add New...」** → **「Project」**
2. リポジトリ一覧から `serp-analyzer` を探す
3. 横の **「Import」** ボタンをクリック
4. プロジェクト設定画面が出る:
   - **Framework Preset**: 自動で「Other」が選ばれます → そのままでOK
   - **Build Settings**: 触らなくてOK
   - **Environment Variables**: 何も追加しなくてOK
5. 一番下の **「Deploy」** をクリック

→ 1〜2分でデプロイが完了します

### 3-3. 公開URLを取得

1. デプロイ完了画面で、**「Visit」** ボタンをクリック
2. 表示されたURL（例: `https://googlesearch-abc123.vercel.app`）が**公開URL**です

> ⚠️ この段階ではまだAPIキー未設定なので分析を実行するとエラーになります。次のステップ4でAPIキーを登録してください。

---

## ステップ4: Claude APIキーをVercelに登録（所要3分）

AIによるキーワード抽出を有効にするため、Anthropic APIキーをVercelに登録します。

### 4-1. Vercelプロジェクト画面を開く

Vercelのダッシュボードから `googlesearch` プロジェクトをクリック。

### 4-2. Settings に移動

画面上部のタブから **「Settings」** をクリック。

### 4-3. 環境変数の設定画面へ

左サイドバーから **「Environment Variables」** をクリック。

### 4-4. 環境変数を追加

入力欄が出てくるので、以下を入力します。

| 項目 | 入力内容 |
|---|---|
| **Key** | `ANTHROPIC_API_KEY` ← 大文字小文字も含めて正確に |
| **Value** | RYOさんのClaude APIキー（`sk-ant-` で始まる文字列） |
| **Environments** | **Production**, **Preview**, **Development** すべてにチェック |

**「Save」** をクリック。

### 4-5. 再デプロイ（重要）

環境変数を追加しただけでは反映されません。**再デプロイ**が必要です。

1. 画面上部のタブで **「Deployments」** をクリック
2. 一番上のデプロイメント行の右側にある **「⋯」**（3点メニュー）をクリック
3. **「Redeploy」** を選択
4. 確認画面で **「Redeploy」** ボタンを押す
5. 1〜2分待つ

### 4-6. 動作確認

デプロイ完了後、公開URLを開いて競合サイトのURLで分析を実行。
結果画面の下部に **「抽出方法: Claude AIによる推定」** と表示されればOKです。

---

## 使い方

1. 公開URLを開く
2. 競合サイトのURL（例: `https://example.co.jp`）を入力
3. 「分析開始」を押す
4. 20〜40秒待つと結果が出る

「複数URL比較」タブでは、最大5サイトをまとめて比較できます。

## トラブルシューティング

### Q1. デプロイに失敗した

`package.json` がリポジトリに含まれているか確認してください。
含まれていれば、Vercelの管理画面で「Redeploy」ボタンを押すと再実行されます。

### Q1-2. 「ANTHROPIC_API_KEYが設定されていません」エラーが出る

ステップ4の環境変数登録が完了していないか、登録後の再デプロイがまだです。
ステップ4-5の再デプロイ手順を実行してください。

### Q1-3. 結果画面に「抽出方法: HTML解析（AIフォールバック）」と出る

AI抽出に失敗してHTML解析にフォールバックしている状態です。原因は3つ:
- APIキーが間違っている、または期限切れ → Anthropicのコンソールで確認
- APIの利用残高が不足 → クレジット追加
- 一時的なネットワーク障害 → 数分後に再実行

「AIエラー:」の後にエラー内容が出るので、参考にしてください。

### Q2. 分析を実行したら「サイト取得失敗」エラー

そのサイトがアクセス制限をかけている可能性があります。別のサイトで試してみてください。

### Q3. 全部のキーワードが「圏外」になる

3パターン考えられます:
- そのサイトがまだSEO対策をしていない（=新しいサイト、内容が薄い）
- 抽出したキーワードがそもそも検索されていない
- Google側で一時的にブロックされている → 5分待って再実行

### Q4. 「結果が一部欠けています」と表示される

Googleが「機械的アクセスかも」と判断して一部結果を返さなかった状態です。
30分〜1時間後にもう一度試すと改善します。

### Q5. もっと頻繁に分析したい

無料スクレイピング方式は1日20〜30サイトが安全圏です。
それ以上必要なら、ScraperAPI（月1000回無料）への切り替えをご相談ください。

## カスタマイズ

### 分析するキーワード数を変えたい

`api/analyze.js` の冒頭にある以下の行を編集:

```javascript
const MAX_KEYWORDS_TO_CHECK = 8;  // ←この数字を変える
```

数字を増やすほど分析が詳しくなりますが、時間も長くなり、ブロックされやすくなります。
推奨: 5〜10。

### 結果を社外秘にしたい

Vercelの管理画面 → プロジェクト → Settings → Deployment Protection で、
パスワード保護やGitHubアカウント認証を設定できます（Vercelの有料プラン要：月$20）。

無料で済ませる場合は、URLを「推測されにくい長い文字列」にすることで簡易的に守れます:
プロジェクト → Settings → Domains で別ドメイン名を設定可能。
